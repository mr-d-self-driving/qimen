# 子平真诠正格样本失败归因与产品方案

> 日期：2026-06-15
> 范围：本轮新增 `zp_*` 22 个教材样本
> 目标：解释当前规则链路与《子平真诠》取用之间的 gap，并给出可实现的产品修复方案。
> 原则：只使用教材中明确给出的取格、成败、旺衰、喜忌和用神结论；未明示字段不作为硬标签扩展。

## 一、总体结论

本轮新增样本不是稀疏杂格，主要是《子平真诠》的正格体系：正官、财、印、食神、七杀、伤官、建禄月劫。当前引擎在调候为急、印星救主、部分特殊形象上已有专项修复，但对《子平真诠》的“正格成败救应”仍偏弱。

新增 22 例中，高频失败并不是单点十神换算错误，而是当前链路缺少一个位于“月令取格”和“五神评分”之间的结构层：

```text
当前：
月令/合化取格 -> 调候 -> 扶抑 -> 格局辅神泛化加分 -> 五神分层

教材实际需要：
月令本气/透干/合会/杂气候选
  -> 正格成格/败格/救应/取清
  -> 输出格局主轴、可用神、失用神、旺衰修正、调候降权
  -> 再进入五神评分
```

因此本轮的核心缺失是：

1. **月令取格过早被合化覆盖**：合化直接改写月令主气，导致正官格、杂气正官、七杀格、财格等被改成正印格、食神格、偏印格。
2. **合会只影响格局名，不回灌旺衰**：金状元“亥卯未三合、官化为印”被用于显示正印格，但旺衰仍按丁火亥月失令，未判身旺印重。
3. **成败救应缺失**：财印并用、财印合失用、合杀留官、财去官清、财格佩印、食印不碍、伤官制杀、杀印有情等教材判断没有进入规则层。
4. **调候与泛扶抑压过格局取用**：调候神常被推成 top1，但教材有时明确说“调候不可为用”或“财已合去失用”。
5. **五神分层缺少“失用/保留/降权”闸门**：被合去、被制、党杀、破印的十神仍按普通扶抑或调候得分；教材明确可用的财、食、官又可能被身弱扶抑扣成忌。
6. **TWO_QI_IMAGE 作为显示候选污染格局解释**：很多正格案例出现“木土成象/火木成象”等描述，虽未覆盖喜忌，但会干扰格局展示和方法命中。

## 三、核心缺失模块

### 1. 正格候选生成器，而不是单一 geju

当前 `getGeJu()` 返回单一格局，且月支合化可以直接覆盖月令本气。新增样本说明应保留多个候选：

- `base_month_pattern`：月令本气取格，如申月庚官、亥月壬官。
- `revealed_hidden_pattern`：月令藏干透出，如未中乙官透。
- `merge_transform_pattern`：三合/六合后的化格，如亥卯未化木。
- `mixed_official_pattern`：官杀混杂、合杀留官、去官留杀。
- `graveyard_pattern`：辰戌丑未杂气财官印透干会支。

产品输出不应只显示一个 `geju`，而应显示：

```js
pattern_candidates: [
  { type: 'base_month', name: '正官格', evidence: ['申月庚金主气'] },
  { type: 'merge_transform', name: '化官为印', evidence: ['亥卯未三合木局'] }
],
final_pattern: {
  name: '正官格',
  subtype: '化官为印透财',
  status: 'formed_with_rescue'
}
```

### 2. 成败救应规则层

新增一个 `BaziPatternResolution` 层，放在调候/扶抑评分之前，专门输出结构效果：

```js
{
  pattern: '正官格',
  subtype: '财印并用',
  status: 'formed',
  useful: ['正官'],
  favorable: ['正印', '正财'],
  invalidated: [],
  unfavorable: ['伤官'],
  strengthAdjustment: 0,
  tiaohouPolicy: 'support_only'
}
```

首批规则按教材样本覆盖：

| 正格 | 首批规则 |
| --- | --- |
| 正官格 | 财印并用、杂气正官、遇伤佩印、合杀留官、官格用印、财去官清、化官为印 |
| 财格 | 财旺生官、财格佩印、用食兼印、财用伤官、财带七杀合杀存财 |
| 印格 | 印格用官、印多用财 |
| 食神格 | 食神生财、食神制杀、食神忌印/透财解印 |
| 七杀格 | 食神制杀、七杀用印、杀印有情、身重杀轻借财清格 |
| 伤官格 | 伤官用财、伤官佩印、伤官带杀、伤官用官（金水独宜） |
| 建禄月劫 | 建禄用官、建禄用财、建禄用杀、禄劫泄秀 |

### 3. 旺衰回灌

当前合会和成格多停在格局展示，未影响旺衰。需要把结构结果回灌到 strength：

- `化官为印`：若三合印局成立且印透，提升日主得助/得地，允许由身弱改为身旺印重。
- `官化伤/官化印`：转换后的五行应进入势能统计，不只改格局名。
- `食神生财`：若教材结构明确“身强食旺财透”，需要检查根气/比劫/食财流通，避免调候印误夺食。
- `财旺身弱`：财官合压日主且印透有根时，直接进入“用印不用官”。
- `比强财轻`：比劫过旺而财有根，允许财用伤官化劫生财。

### 4. 十神失用/保留闸门

新增 `shen_status`，让十神不只有分数，还能被标记：

```js
shen_status: {
  正财: { state: 'favorable', reason: '财印两不相碍' },
  偏财: { state: 'invalidated', reason: '丁壬合，财印两失' },
  七杀: { state: 'invalidated', reason: '乙庚合杀留官' },
  食神: { state: 'protected', reason: '食神制杀，财不可通关生杀' }
}
```

关键闸门：

- **合去失用**：丁壬合、乙庚合等，若教材结构判“财印两失/合杀留官”，被合之神不再靠调候或扶抑上位。
- **财印不碍**：隔位或中隔日主时，财不破印，不能按身弱普通扶抑直接判忌。
- **财党杀**：七杀食制时，财星若生杀，应降权；调候财不得覆盖制杀用神。
- **印夺食**：食神生财/食神制杀成立时，印星不得因调候或身弱扶抑夺食。
- **官杀取清**：合杀留官/去官留杀后，被去的一方降为失用。

### 5. 调候降权策略

调候仍重要，但需根据格局状态分层：

| 场景 | 调候角色 |
| --- | --- |
| 无明确成格取用 | 可作为主用 |
| 格局主矛盾明确，如食神制杀、伤官制杀 | 只能作辅助或风险提示 |
| 调候神会生忌神，如财生杀 | 降权，甚至入忌 |
| 化官为印后身旺印重 | 调候印不得继续加主用分 |

建议新增：

```js
tiaohouPolicy:
  'PRIMARY_ALLOWED' |
  'SUPPORT_ONLY' |
  'RISK_HINT_ONLY' |
  'DISABLED_BY_PATTERN'
```

### 6. TWO_QI_IMAGE 展示与覆盖拆分

新增样本里很多正格被附上“木土成象/火木成象/金水成象”。这些未必覆盖喜忌，但会污染格局解释。

规则：

- 未达到 override 的形象只进入 `image_analysis.candidates`，不拼进 `actual.geju/chengge/pattern_final`。
- 正格成败救应已成立时，普通两气候选默认 `DESCRIPTIVE_ONLY`，不参与格局评分。
- 若 TWO_QI 要覆盖，必须满足“教材命名形象/专旺/从格/两气成象”白名单或高置信 override。

## 四、产品方案

### Task 1：新增正格解析层

新增模块建议：

- `lib/baziPatternResolver.js`
- 输入：四柱、十神、藏干、透干、合冲刑害、三合六合、旺衰初判、调候初判。
- 输出：

```js
{
  basePattern,
  finalPattern,
  candidates,
  patternEffects,
  decisionMessages
}
```

`patternEffects` 最低需要：

- `promoteShens`: 主用/喜神加分。
- `demoteShens`: 忌神/失用神降权。
- `protectedShens`: 不可被同五行折叠误伤。
- `invalidatedShens`: 合去、破格、党杀、夺食。
- `strengthAdjustment`: 成局对旺衰的修正。
- `tiaohouPolicy`: 调候是否可上位。

### Task 2：先实现 6 类高收益规则

优先规则按影响面排序：

1. **月令本气保留 + 合化候选化**
   修薛相公、金状元、李参政等格局被覆盖问题。

2. **财印关系规则**
   - 财印并用：薛相公。
   - 财印合失用：无名杂气正官。
   - 财格佩印：曾参政、伍朝枢。
   - 印多用财：汪侍郎、金状元。

3. **官杀取清规则**
   - 合杀留官：李参政。
   - 财去官清/制伤官清：范太傅。
   - 官杀混杂取清。

4. **食伤制杀规则**
   - 七杀食制：谭延闿。
   - 伤官带杀用伤制杀：史量才。
   - 财生杀降权。

5. **食神生财/伤官用财规则**
   - 梁丞相、谢阁老、史春芳、汪学士。
   - 防止调候印夺食。

6. **建禄月劫取用规则**
   - 金丞相：建禄用官，印护官。
   - 官星透出优先于调候财。

### Task 3：重排五神评分

建议分数来源从“并列加法”改成“结构层先判主辅，再加权”：

```text
硬结构主用神：+60
硬结构喜神：+35
硬结构忌神/失用：-60
调候主用：+50，但受 tiaohouPolicy 限制
扶抑：+/-20
普通格局辅神：+15~20
```

若 `invalidatedShens` 命中，即使有调候/扶抑加分，也不得进入 top1。

### Task 4：前端呈现

新增“格局成败”解释段，不只显示格局名：

```text
基础格局：正官格
成败救应：财印并用
成格依据：申月庚金官星秉令；壬印戊财，中隔日主乙木，两不相碍。
取用结论：正官为用，正印与正财为喜；火伤官为忌。
```

对于合化：

```text
基础格局：正官格
格局变化：亥卯未三合，官化为印
旺衰修正：印局成势，身旺印重
取用结论：用庚金财星损印，喜食伤财，忌再增木水。
```

### Task 5：评测验收

验收必须跑：

```bash
node eval/baziprofile-accuracy/run.mjs
node eval/baziprofile-accuracy/report.mjs
```

重点验收新增 `zp_*`：

| 验收组 | 目标 |
| --- | --- |
| 格局命名 | `zp_001/002/003/005/014` 不再被合化直接改成错误主格 |
| 旺衰回灌 | `zp_003/013/016/018/020/021` 至少不再因明显结构反判身强身弱 |
| 财印规则 | `zp_001/002/003/007/011/017` 喜忌方向不反 |
| 制杀规则 | `zp_008/009/019` 不再由调候财/食伤错误上位 |
| 食伤生财 | `zp_013/016/018/021` 食伤财进入喜用方向，印不得反夺 |
| 建禄用官 | `zp_022` 官印进入 top1/top2，调候财降为辅助 |

| 汪侍郎 | 金白水清覆盖，正印 top1 | 印多用财，财星上位，金印不过度为喜 |
| 金丞相 | 建禄格命中但用财 | 建禄用官印护，官印进入 top1/top2 |

## 六、CP / 需要确认点

1. **合化是否允许覆盖月令取格**
   建议改成“默认生成变化候选，不直接覆盖基础正格”；只有教材形象/化格明确成立时，才作为 `finalPattern`。

2. **《子平真诠》正格是否作为通用引擎主线**
   这批不是稀疏格，建议纳入主线准确率，不放入 `classical_pattern`。

3. **调候权重是否必须受格局状态约束**
   建议是必须。否则谭延闿、史量才、金状元这类会持续被调候反向带偏。

4. **前端是否展示“基础格局 + 成败救应”双层**
   建议展示。单一“格局名”不足以表达财印并用、合杀留官、化官为印这类教材取法。

## 七、与其他教材体系的兼容性

结论：上述方案不是只针对《子平真诠》新增 case 规则，而是把现有链路拆成“月令基础格局、结构变化候选、成败救应、旺衰回灌、调候策略、喜忌裁决”六层。这个拆分可以兼容目前材料中的三条主线，但需要明确不同体系的优先级边界。

### 1. 《子平真诠》格局法：高度兼容，且应作为正格主线

《子平真诠》的核心不是只取一个格局名，而是围绕月令用神继续判断成格、败格、救应、清浊、有情无情。例如：

- 薛相公案：正官格基础成立，但关键在“壬印戊财，以乙隔之，水与土不相碍”，所以不是简单“正官 + 正印 + 正财”并列加分，而是财印并用且互不破坏。
- 无名杂气正官案：杂气正官有官透会支的结构，但“壬财丁印相合”导致财印两失，所以应保留正官结构，同时标记孤官无辅/财印失用。
- 金状元案：亥卯未三合后“官化为印”，后续判断转为身旺印重、用财损印，而不是仍按普通身弱取印。

因此，方案中的 `BaziPatternResolver`、`successFailureState`、`invalidatedShens`、`patternEffects` 与《子平真诠》兼容，而且是当前引擎缺失的主干能力。

### 2. 《穷通宝鉴》调候法：兼容，但不能无条件抢主用

《穷通宝鉴》重视月令寒暖燥湿、天干透藏和调候先后，材料中多处强调取用要看成败得失、透藏、有情无情，也承认“取用无定法”。这与“调候仍保留，但受格局状态约束”的方案兼容。

产品层需要把调候分成三种状态：

| 状态 | 含义 | 适用 |
| --- | --- | --- |
| `PRIMARY` | 气候病急，调候可直接主用 | 寒暖燥湿极端，且无强成格结构反向约束 |
| `SUPPORT_ONLY` | 调候只作辅助喜神/风险提示 | 已有食神制杀、财印并用、化官为印、建禄用官等硬结构 |
| `DISABLED_FOR_YONG` | 调候不参与主用竞争 | 调候元素会破坏教材中的成格方向，或已被合冲刑害标记失用 |

这可以解释现有 gap：谭延闿、史量才、金状元等不是“不需要调候”，而是调候不应越过已成的格局取用；朱元璋、董中堂这类气候/湿燥确实重要的案例，仍应保留调候诊断。

### 3. 《滴天髓》/陆致极气势法：兼容，但要走候选覆盖门

气势法、从旺、从儿、两气成象等材料强调五行气势、清浊、顺逆、可破不可破。它与正格法不是同一层判断，因此不能直接混进月令正格，也不能只按元素占比覆盖。

兼容方式：

- 先生成 `imageCandidates`，包括从旺、从儿、两气成象、专旺等。
- 再判断清纯度、主气是否成势、是否被关键冲克合化破坏、是否有教材命名形象。
- 只有通过覆盖门的形象，才覆盖普通旺衰/正格主线。
- 未通过覆盖门的形象，只作为“形象提示”展示，不改写喜忌和用神。

这与前面两气成象讨论一致：朱元璋、张謇、冰心这类需要形象覆盖；康熙这类即使识别成专旺，喜用方向和普通格局一致，影响较小；部分三命通会稀疏格不应纳入主线覆盖。

### 4. 《三命通会》稀疏格法：部分兼容，应独立降权

《三命通会》里的魁罡、官印双全、印绶天德等有独立取法，关键条件稀疏，和《子平真诠》正格主线并不完全同构。当前评测中已确认这类 case 不应作为普通准确率的主线修复目标。

产品上建议保持独立层：

```text
classicalPattern:
  detected: true
  family: SANMING_SPECIAL
  displayOnly: true / advisory
  overrideYongXi: false by default
```

只有材料同时给出明确取用方向，且不与正格/调候/气势主线冲突时，才允许参与喜忌权重。

### 5. 兼容后的架构边界

| 教材体系 | 引擎对应层 | 是否覆盖格局 | 是否覆盖旺衰 | 是否覆盖喜忌用神 |
| --- | --- | --- | --- | --- |
| 子平真诠正格 | 基础格局 + 成败救应 | 是，作为主线 | 可回灌 | 是 |
| 穷通宝鉴调候 | 调候策略层 | 通常不覆盖 | 通常不覆盖 | 可主用/辅助/禁用，取决于气候病急与格局冲突 |
| 滴天髓气势/形象 | 特殊形象候选层 | 通过覆盖门后覆盖 | 通过覆盖门后覆盖 | 通过覆盖门后覆盖 |
| 三命通会稀疏格 | 古籍特殊格提示层 | 默认不覆盖 | 默认不覆盖 | 默认不覆盖 |

### 6. 对当前方案的修正

因此，后续实现不能写成“新增若干子平真诠 hardcode 规则”。更稳的方案是：

1. `getGeJu()` 只产出基础月令格局，不被三合六合直接改写。
2. 新增 `derivePatternCandidates()`，把合化、墓库透干、财印关系、制化关系作为候选结构。
3. 新增 `resolvePattern()`，按教材可解释的成败救应选择最终结构。
4. 新增 `applyPatternEffects()`，把合化成势、印重、财印失用、杀被制、食伤生财等结构效果回灌到旺衰和十神状态。
5. 调候从固定加权改为 `tiaohouPolicy`，与格局状态联动。
6. 前端展示“基础格局 / 结构变化 / 成败救应 / 调候影响 / 取用结论”，避免用户只看到一个格局名却看不见推导链。

这套边界可以兼容其他教材：正格、调候、气势分别保留自己的判断入口，但最终喜忌用神必须由统一 resolver 做冲突裁决。

### 2. 对架构的新增要求

80 例复评后，原方案仍成立，但需要把“成败救应”从格局命名扩展到十神状态机。建议新增或明确以下状态：

```ts
type ShenStatus =
  | 'PRIMARY_YONG'      // 主用
  | 'ASSISTANT_XI'      // 相神/喜神
  | 'PROTECTIVE'        // 护格、护官、护印、护财
  | 'TRANSFORMED'       // 合化/会局后变性
  | 'INVALIDATED'       // 合去、冲破、失用
  | 'DISEASE'           // 病神
  | 'NEUTRAL';
```

关键不是给十神简单加分，而是先由 `resolvePattern()` 生成结构性裁决：

```ts
{
  basePattern: '财格',
  variant: '制杀生财',
  primaryYong: ['食神'],
  assistantXi: ['财星'],
  protectiveShens: ['官星'],
  invalidatedShens: ['失用之印', '被合之财'],
  tiaohouPolicy: 'SUPPORT_ONLY'
}
```

然后 `applyPatternEffects()` 再把这些状态回灌到五神评分：

- `PRIMARY_YONG` 不得被普通扶抑或调候反超。
- `ASSISTANT_XI` 不得被机械扶抑打入忌方。
- `INVALIDATED` 即使五行适合，也不得进入 top1。
- `TRANSFORMED` 必须改变后续旺衰与喜忌，例如官化印、伤化财、劫化财。
- `PROTECTIVE` 至少应进入喜用方向，除非被另一个更高优先级结构破坏。

### 1. “取清”与“去留”必须成为结构规则

新增《子平真诠》样本中，多例不是简单“七杀格/正官格/伤官格”命名，而是依靠去留取清成格：

| 规则 | 代表 case | 当前表现 | 需要的结构效果 |
| --- | --- | --- | --- |
| 煞用食制，不要露财透印 | 无名极等贵命 | 识别七杀格，但取偏印且落入书中忌向 | 食神设为 `PRIMARY_YONG`；财、印在该结构下标为 `DISEASE/INVALIDATED` |
| 七杀用财，去印存食 | 周丞相 | 取劫财，伤官土入忌，印反入喜 | 财为用，食伤为相；印为被去之物 |
| 去官留杀 / 去杀留官 | 岳统制、沈郎中 | 月令格局被普通取格或两气提示带偏 | 官杀并见时，resolver 必须按月令主轴和合冲去留裁决最终官/杀 |
| 七杀无食用印 | 赵员外 | 七杀上位，火印未成主用 | 无食制杀时允许印化杀为用 |
| 官杀竞出取清 | 无名阳刃命 | 羊刃格命中但不能表达壬合丁、留杀取清 | 标记被合官、保留杀，食神/合神参与结构 |
| 去伤存官 | 王总兵 | 建禄格命中，但庚印入忌 | 庚合乙去伤，庚应为护官相神而非忌 |

这说明 `resolvePattern()` 不能只返回“格局名 + 主用神”，还要返回“被去掉的忌神”和“被保留的清神”。建议新增：

```ts
{
  retainedShens: ['七杀'],
  removedShens: ['正官'],
  cleaningAction: 'remove_officer_keep_killing',
  cleaningBy: ['合', '冲', '制']
}
```

### 2. 伤官变化需要独立 variant

新增伤官样本进一步证明，伤官格不能只按“逆用印绶/财星”概括。

| 伤官 variant | 代表 case | 当前 gap |
| --- | --- | --- |
| `output_transforms_to_wealth` | 罗状元 | 被普通财格/印星带走，未识别“化伤为财” |
| `output_wealth_affectionate` | 秦龙图 | 财伤有情未识别，伤官/财入忌 |
| `output_with_wealth_and_seal` | 都统制、无名丞相 | 用印可命中一半，但财或印的另一半常被判忌 |

这部分仍属于原架构的 `patternEffects`，但实现时要把“变化”作为一等结构，而不是只在 method 文案里描述。

### 3. 从格 FOLLOW_IMAGE 需要补 subtype 与覆盖

新增《滴天髓》从象样本暴露了另一个边界：当前特殊形象层已经能识别部分 FOLLOW_IMAGE，但 subtype 与覆盖还不稳定。

| Case | 当前表现 | 目标 |
| --- | --- | --- |
| 无名状元命：从财格 | 被判正财格/火土成势，用神偏向普通财格 | 明确 `FOLLOW_IMAGE/从财格`，喜火土，丙火秀气入用 |

```ts
followImage: {
  subtype: '从财格' | '从杀格' | '从儿格' | '从势格',
  overrideNormalPattern: true,
  favorableElements: [...],
  unfavorableElements: [...]
}
```

- 最终格局展示
- 旺衰口径：极弱/弃命从势
- 喜忌方向：顺所从之势
- 用神：优先取引通/泄秀/生财之神，而不机械取印比

## 十、基于黄金测试集的数据归因与推荐优化次序（Claude 复核，2026-06-15）

> 本节不是新增方案，而是把上面的方案对着 `results/latest.json`（100 例实测）和现有代码逐条核验后，给出**按数据杠杆排序的落地次序**。结论：原方案方向对，但优先级应重排——最大的一块是“成格已经算对、却没接进评分”，属于接线问题，不是规则缺失。

### 2. 关键发现：成格算对，但用神被丢弃

对“用神完全判错（yong ratio=0）”的案例做拆分：

- **41/100** 案例 chengge 已识别，但用神全错；
- **24/100** 案例 geju 完全正确，用神仍全错。

把这 41 例再拆成两类：

| 类 | 数量 | 含义 | 代表 |
| --- | ---: | --- | --- |
| **A：成格已解出，但用神被评分丢弃** | 11 | `getChengGe` 已得出教材用神，却没进评分 | 袁树珊[杀邀食制格]、谭延闿[杀邀食制格]、谢阁老[食神生财格]、李御史/林尚书/王太仆[用财喜比格]、章丞相[财旺生官格]、贾平章[刃杀相停格] |
| **B：成格根本没解出（落默认格名）** | 30 | 合化覆盖 / 透干误选 / 变体规则缺失 | 金状元[误成正印格]、梁丞相[食神生财却判伤官格]、汪侍郎[偏印格]、史量才[伤官格未带杀] |

**根因已定位到代码**：[baziCore.js:433](../../lib/baziCore.js) 的 `getChengGe` 算出了 `yongShenTenGod` + `xianShen`（即教材用神/相神），但 [baziCore.js:1223](../../lib/baziCore.js) 调 `BaziRuleEngine.getFavorableUnfavorable(...)` 时**没有传 `chengGeDetail`**。成格用神只用于展示（`buildPatternAnalysis`），完全没参与 `five_shens` 评分。评分仍是纯加法：调候 50/35（`getTiaohouWeight`）+ 扶抑 ±20 + 格局辅神 15~20，top1 即用神，无失用闸门。所以袁树珊“杀邀食制”算对了，用神却被 L1 调候的癸水（偏印）顶上去。

### 3. 推荐优化次序（替代第四章 Task 1–5 平铺）

按“数据杠杆 ÷ 爆炸半径”排序：

**P0-a　把 `chengGeDetail` 接进评分（最高性价比）**
- 改动：`getFavorableUnfavorable` 增参 `chengGeDetail`；当 `chengGeResult==='成格'` 时，把 `yongShenTenGod` 设为强先验（用神硬底分），`xianShen` 进喜方向；`'败格'` 时把对应十神标 `INVALIDATED`，不得进 top1。
- 直接命中 A 类 11 例；并对 xiji（43 例最大头）有外溢收益。
- 爆炸半径小：单函数签名 + 评分先验，不动旺衰、不动格局识别。**先做这一刀并单独量一次。**

**P0-b　修 line 934 合化覆盖 → 候选化**
- `getGeJu` 当前 `monthMergeInfo` 直接改写 `monthMainGan`（[baziCore.js:934](../../lib/baziCore.js)），导致 B 类里金状元、无名杂气正官、汪侍郎落错格。改为默认产候选、不覆盖基础正格。
- 解锁 B 类中“格名就错”的子集。

**P0-c　`tiaohouPolicy` 四态**
- 已有成格时调候降为 `SUPPORT_ONLY`，治调候偏印/正印顶用神（袁树珊、谭延闿、谢阁老的枭夺食）。低风险高收益。

**P1　变体补全（仅 B 类残余需要）**
- A 类靠接线即可，不需要变体。B 类里 chengge 落默认格名的（史量才伤官带杀、葛参政财旺生官、梁丞相食/伤误判）才需要第八/九章的 variant。**在 P0 三刀量到泛化收益后再铺，避免 30 个 variant 过拟合金样本。**

**P1　旺衰回灌 / 评分重排（最后做，带护栏）**
- strength 已 69%，是最好维度；回灌有回归风险。必须 feature-flag + 对 55 个 strength case 做回归后再合。

### 31. 遗留问题同源分析与迭代方案更新（2026-06-16，Claude）

把 §15/§24/§30 的遗留与当前 129 例失败一起做根因相关性分析，结论：**遗留问题大面积同源，主根是「引擎系统性过判身弱」。**

**数据证据**：

| 观察 | 数值 |
| --- | --- |
| gold 身强但引擎判身弱（过判弱） | **18** |
| gold 身弱但引擎判身强（过判强） | 3 |
| 方向一致 | 62 |

| 过判弱 18 例中 xiji 也错 | 12 |

**同源机制（一条因果链解释大部分遗留）**：

```
过判身弱  →  L3 扶抑机械抬印比、压食伤财官  →  成格用神(食/伤/财/官)被印比反夺  →  yong/xiji 错
```

- 之前当"身弱判对、成法该优先"处理（窄保护），但数据显示**很多身弱本身就是误判**——上游修对，下游一半补丁都不必打。

**重分类**：

| 簇 | 例数 | 根因 | 关系 |
| --- | ---: | --- | --- |
| 过判身弱 → 扶抑压成法 | ~12–18 | strength 偏弱 | **主根（同源）** |
| 从格/专旺未识别 | ~9 | 极弱应从但 FOLLOW 评分/类别竞争不足 | 旺衰同族（极端弱端）|
| 喜/用/忌 三态无法表达 | 贯穿 | effects 只能加减分 | 下游补丁层症状 |
| 复合救应（范太傅/周丞相）| 2 | resolver 复合 | 独立少数 |

前两簇其实是同一根光谱的两端：**旺衰判断在"偏弱↔身强"和"极弱↔从格"两处都偏向了错误一侧。**

**迭代方案更新（取代原"旺衰回灌放最后做"的排序）**：

- **P3-a（新最高优先）：旺衰校准，修过判身弱。** 上游、同源、不对称方向明确（提高身强敏感度）。预计直接修复 strength 18 例 + 级联 yong/xiji 8–12 例，杠杆远超单条成法变体。

- **P3-c：FOLLOW 评分 + TWO_QI/FOLLOW 类别竞争重做。** 依赖 P3-a（旺衰极弱端判准）与 P3-b（三态）。

### 1. P3-e 执行设计确认（2026-06-18，Codex）

#### P3-e1：藏支印比对“得助”的有限回灌

**目标**：修正 `getSupportDetail()` 只统计透干印比、完全忽略藏支印比实助造成的远端过判弱；不改变藏支用神选择，不调整 `getStrengthBand()`。

**设计边界**：

- 藏支印比只作为“得助补充”，不得重复计入 `getRootDetail()` 已计算的日主同五行根气。
- 只承认本气/中气中的印比；余气不进入第一刀，避免微根全局抬分。
- 采用低权重、封顶式补分；月支、日支仍按现有柱位权重区分，但补分总额不得抬高 `support_detail.max_score`。

- 输出单独的 `hidden_supports` 与 `hidden_support_score`，让逐例 diff 能区分透干实助和藏支补助，不把新分数藏进旧字段。

**测试顺序**：

1. 先写单测锁定“藏印有补助、同源比劫不双算、余气不计、补分不越上限”。

#### 21.3 与 §31 / §37 路线的关系

§31 定位的同源主根是“旺衰偏弱 + 藏支/泄秀用神不足”。这批 PDF fixture 继续验证该判断，但暴露出另一条更强的体系差异：**穷通调候专用干不是普通扶抑喜忌，也不是子平正格成败救应，它需要独立的月令病药 resolver。**

因此 P4 不应继续在现有五神分数上直接加权，而应把入口拆成三层：

```text
月令体系 resolver
  ├─ ZiPingPatternResolver：月令正格、成败救应、取清、相神状态
  ├─ QiongTongResolver：日干 + 月令 + 寒暖燥湿 + 透藏病药专用干
  └─ ImageResolver：专旺 / 从格 / 两气成象的最终覆盖与 display-only 候选

三层输出 patternEffects / shen_status / tiaohouPolicy
  -> 再进入 five_shens 聚合
```

关键边界：

- 《子平真诠》样本按正格成败救应走 `ZiPingPatternResolver`。
- 《穷通宝鉴》明确“专用某干/以某干为用/无某则某”时，走 `QiongTongResolver`，不能被普通扶抑直接覆盖。
- 专旺/从格若达到完整 frame，应由 `ImageResolver` 输出最终覆盖；未达完整 frame 的 TWO_QI 只 display。
- methodTag 只补解释链，不作为评分硬先验，防止“为了命中原词”污染判断。
