# 奇门遁甲评分引擎改进计划

> 基于「不第一届奇门遁甲预测培训资料汇编」+ GitHub 教材审查结果
> 创建日期：2026-05-19
> 文件路径：`lib/qimenScoringEngine.js`

---

## 背景

当前评分引擎分数过度集中于 40–70 区间，无法有效区分强吉盘与大凶盘。
根因分析：旺衰权重偏低、九星权重不足、缺失有名格评分、缺失主客动静维度。

**教材核心结论（培训资料原文）：**
- "天时影响事物的力量**占综合力量的50%以上**"（星 = 天时）
- "事物的成败是由能量决定，而不是符号……旺衰具有**一票否决权**"
- "三奇得使虽为吉格，在事物大凶时只能稍向好的方向转化，不至大凶——绝不能直接转大吉"
- "得吉门比得三奇还重要"

## P1：有名格全局层（产品方案）

### 架构决策：新建全局层，整体移除宫位层 scoreFormation

#### 为什么要整体移除 `scoreFormation`

当前 `evaluatePalace` 内的 `scoreFormation` 是按宫位计算的天地盘干组合得分（±3），其中已包含若干有名格的简化代理：

| scoreFormation 当前条目 | 对应有名格 | 旧分值 | 新全局分值 | 问题 |
|------------------------|-----------|--------|-----------|------|
| `戊+丙` | 青龙返首 | +3 | +10 | 严重低估，且双计 |
| `乙+辛` | 青龙逃走 | -3 | -12 | 严重低估，且双计 |
| `辛+乙` | 白虎猖狂 | -3 | -12 | 严重低估，且双计 |
| `丙+庚` | 荧入太白 | -3 | -8 | 低估，且双计 |
| `庚+癸` | 大格 | -3 | -10 | 低估，且双计 |
| `丙+辛` | 奇仪相合（丙辛合）| +3 | → 移入全局层 +4 | |
| `乙+丙` | 日月并行 | +3 | → 移入全局层 +5 | |
| `丁+壬` | 奇仪相合（丁壬合）| +3 | → 移入全局层 +4 | |

**结论**：将 `scoreFormation` 整体从 `evaluatePalace` 移除，所有天地盘干组合的评分统一收入新的 `evaluateNamedFormation` 全局层。

### 双加双扣风险完整分析

| 有名格信号 | 与哪层可能重叠 | 是否真实重叠 | 解决方案 |
|-----------|--------------|------------|---------|
| 天地盘干（sky+earth）| **宫位层 scoreFormation** | ✅ **有** | 移除 scoreFormation |
| 天地盘干（sky+earth）| **极性翻转层（polarityRules）** | ✅ **有**（乙+辛、庚+丙、丙+庚 三条已在 polarityRules 中）| 有名格命中时，对应 polarityRules 条目跳过不计入 contextDelta |
| 天地盘干（sky+earth）| **宫位层 门/星/神**（门克宫、星凶、神煞）| ❌ **无** | 信号不同，不重叠（干 vs 门/星/神）|
| 天地盘干（sky+earth）| **生克层**（日时五行生克）| ❌ **无** | 生克层用宫位五行，非天干 |
| 天地盘干（sky+earth）| **宏观层**（值符/值使）| ❌ **无** | 宏观层用 god/isZhiShi，非天干 |
| 六仪击刑（宫位刑落）| **宫位层 旺衰**（落刑宫往往衰死）| ⚠️ **部分** | 六仪击刑是结构格，旺衰是能量评估。允许叠加，但击刑得分应取固定值不随旺衰再乘 |
| 五不遇时（时干克日干）| **生克层**（时干克日干 → -14）| ✅ **有** | 五不遇时全局打-10，同时生克层已给-14。五不遇时只记 `layer: named_formation`，生克层照常；不额外扣，但 LLM prompt 要标注双重打压 |
| 三奇入墓（奇临墓宫）| **宫位层 旺衰**（墓宫 ground 已扣-5）| ⚠️ **部分** | 入墓是宫位能量，三奇入墓是奇的状态。允许叠加，因为两层信号角度不同 |
| 门迫（八门克宫）| **宫位层 isDoorPressured**（已在代码中处理）| ✅ **有** | 门迫已由 `isDoorPressured` 在宫位层处理，**不进入全局格层** |

#### 核心规则

> **有名格层命中时，以下操作同步执行：**
> 1. 从 `polarityOverrides` 中过滤掉同 signal 的条目，不计入 `contextDelta`
> 2. 有名格 delta 进入独立的 `deltas.named_formation` 字段
> 3. 不影响宫位层的 门/星/神/旺衰 评分（这些是独立信号）

### 格局领域差异化分值矩阵

**核心原则：** "获得类域"（财/事/情）与"摆脱类域"（健康驱病、被动官司）极性相反。

- **获得类**（career_business / finance_wealth / relationship / exam_study）：吉格加分，凶格扣分
- **摆脱类**（health_action 驱病面）：部分凶格转正（"贼去了 = 病去了"），部分吉格转负（"病深入内"）
- **法律**（lawsuit_legal）：区分主动诉讼（积极追讨）vs 被动受告（防守局）
