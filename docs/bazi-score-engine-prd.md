# 八字运势算分引擎重构 PRD

## 背景与目标

当前运势算分引擎以日、月、年为核心输出节奏判断。重构目标是把原先“喜用 vs 忌仇”的二维模型，升级为“用神、喜神、闲神、仇神、忌神”的分层模型，并依据 NotebookLM 中八字命例材料所强调的判断链路：

排盘口径 → 日主旺衰 → 寒暖燥湿调候 → 月令格局 → 十神组合 → 喜用忌神 → 刑冲合害 → 神煞辅助 → 大运流年应期 → 事件校验。

本轮已优先完成 P0 与 P1，使核心分数不再被旧的二维喜忌、调候顺序、三刑熔断和大运/流年平权逻辑束缚。

## 模块架构

```text
lib/
  normalizeShenProfile.js      四维用神共享工具
  calculateTiaohouRatio.js     调候拦截器
  calculateDailyScore.js       日运引擎
  calculateMonthlyScore.js     月运引擎
  calculateAnnualScore.js      年运引擎
  constants/tiaohou.js         穷通宝鉴调候表
```

## P0 核心体系重构

### P0-A 用神四维体系重构

理论依据：

- 《滴天髓》“用神为药、喜神辅用、忌神为病、仇神助病”。
- NotebookLM 命例材料强调：闲神平时不作主判断，但遇关键合冲时可转义。

已完成：

- 新增 `lib/normalizeShenProfile.js`。
- 支持 `yong_shen / xi_shen / ji_shen / chou_shen`。
- 兼容旧字段：`favorable_elements -> yong`，`unfavorable_elements -> ji`。
- 支持别名解析：如 `印星 -> 正印/偏印`。
- 新增 alias-aware 冲突去重：高优先级层级会剔除低优先级重复命中。
- 日、月、年三套引擎已接入四维层级。

关键权重：

| 层级 | 日运天干 | 月运天干 | 年运天干 |
| --- | ---: | ---: | ---: |
| 用神 | +10 | +12 | 流年 +14 / 大运 +8 |
| 喜神 | +6 | +7 | 流年 +9 / 大运 +5 |
| 闲神 | 0 | 0 | 0 |
| 仇神 | -5 | -6 | 流年 -7 / 大运 -4 |
| 忌神 | -8 | -10 | 流年 -12 / 大运 -7 |

状态：完成。

理论依据：

- 《穷通宝鉴》以调候为急。
- NotebookLM 命例材料强调：调候代表命主生存环境条件，遇常规格局冲突时调候优先。

已完成：

- 新增 `lib/calculateTiaohouRatio.js`。
- 读取 `lib/constants/tiaohou.js` 中 `TIAOHOUS` 表。
- 月运以流月天干查调候，年运以流年天干查调候。
- `score_hits.layers` 增加 `tiaohou` 层。
- 调候乘数已移到空亡、三刑、自刑等 cap 之前，避免调候加成抬穿封顶。
- debug 输出包含 `rawBeforeTiaohou / tiaohouRatio / rawAfterTiaohou`。

系数：

| 情形 | 系数 |
| --- | ---: |
| 满足第一调候用神 | x1.08 |
| 满足次级调候用神 | x1.04 |
| 加剧气候偏枯 | x0.93 |
| 无影响 | x1.00 |

状态：完成。

## P2 神煞旺衰联动

### P2-A 神煞得分与月令旺衰联动

理论依据：

- NotebookLM 材料提炼：“贵人无气，虽有如无”；神煞不可拘定，轻重较量，要在通变。

目标方案：

- 天乙、文昌等吉神不再固定加分。
- 引入 `qiRatio * tierRatio * voidRatio`。
- 吉神落旺相且为喜用才完整加分；落忌仇或空亡应归零或大幅折减。

状态：待开始。

### P2-B 月运 Layer2 权重提升

问题：

- 月运日运聚合层当前区间为 ±5，对月运结论影响偏弱。

目标方案：

- 月运 Layer2 调整至 ±10。
- 优化连续低分日、节气交接、空亡叠加的低谷识别。

状态：待开始。

## P3 细节完善

### P3-A 日运贪合忘冲补全

理论依据：

- NotebookLM 材料强调“贪合忘煞、贪合忘官”可改变凶神破坏力。

目标方案：

- 日运存在六合/三合且同时触发六冲时，冲罚减半。
- 若合走忌神，冲罚免除并小幅加分。
- 若合走用神，标记“贪合失用”，不减半。

状态：待开始。
