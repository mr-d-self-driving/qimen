# Bazi TWO_QI_IMAGE 覆盖逻辑产品方案

> 日期：2026-06-12  
> 范围：Bazi Profile 规则引擎中的 `TWO_QI_IMAGE` 命名两气形象覆盖逻辑  
> 目标：保留已验证的两气形象收益，同时避免有效制衡/制刃用神被 L0 顺势覆盖误杀。

## 背景

当前 `TWO_QI_IMAGE` 会先做五行气势统计，取占比最高的两个五行。普通两气只作为描述性形象，不覆盖下层取用；命名两气形象达到审核阈值后，会进入 L0 特殊气势覆盖。

当前会覆盖的命名两气形象有三类：

| subtype | 当前触发条件 | 覆盖策略 |
| --- | --- | --- |
| 金白水清 | 金水为 top2，月令申/酉，合计占比 >= 0.60 | 喜金水，优先取水泄秀，忌火土 |
| 火土成势 | 火土为 top2，合计占比 >= 0.62 | 顺火土势，喜火土，忌木金水 |
| 强众敌寡 | 土日主，火土为 top2，合计占比 >= 0.62，存在弱敌元素 | 取弱敌为用，同时保留火土强党为喜 |

覆盖后的引擎行为是：

```text
L0 特殊气势
  -> 直接给特殊形象喜忌评分
  -> 跳过调候、扶抑、病药、格局制化、救主等下层逻辑
```

这让冰心、朱元璋、张謇等 case 的喜忌方向归位，但也暴露出一个产品风险：当第三气其实是书中主用神时，L0 顺势会把它误判为逆势忌神。

两气成象不应被理解成“top2 五行占比高就顺势”。更接近教材的通用框架是：

```text
清气成体段 / 两气双肖
  -> 看两气是否生化有情、安放得所
  -> 看旺神是太过还是不太过
  -> 看弱神是有根还是无根
  -> 再决定宜泄、宜克、宜扶、宜去、宜顺
```

### 1. 两气成象首先要“成象”，不是简单统计

NotebookLM 对资料库的归纳指出，两气成象包含相生各半、相克均敌两类；若两行势力并不均等，则需要进一步确定两行成象的“重心”，不能直接按均衡两气处理。

产品含义：

- `combinedRatio` 只能证明两气集中，不能证明两气成象。
- `ratioDifference` 必须参与判定：相克两气尤其要求均敌，偏重偏轻时不应直接覆盖。
- 两行不均等时，应输出 `two_qi_center_of_gravity`，再决定是泄旺、扶弱、去寡，还是顺清势。

### 2. 两气成象还要“清气成体段”

《滴天髓阐微·出身》说：

```text
清得尽者，非必一一成象，虽五行尽出，而能安放得所，生化有情，不混亲神忌客，决发科甲。
清得尽者，非一行成，两气双肖也。
```

因此，两气成象的核心不是只看五行数量，而是：

- 两气是否成一个清晰体段；
- 生化是否有情；
- 闲神、忌客是否不司令、不深藏、不混局；
- 即便有浊气，也要能被安放、制化，不能破坏喜用。

产品含义：`TWO_QI_IMAGE` 的 `combinedRatio` 只能做候选，不足以直接决定覆盖。

### 3. 取用要看“旺神/弱神”的状态

《滴天髓阐微·疾病》给出非常关键的通用取法：

```text
旺神太过者宜泄，不太过宜克；
弱神有根者宜扶，无根者反宜伤之。
凡八字须得一神有力，制化合宜。
```

这比“顺势/逆势”更细：

- 旺神太过：宜泄，不宜强克。
- 旺神不太过：可以克制。
- 弱神有根：可以扶起作为用神。
- 弱神无根：反而宜去，不宜扶。

产品含义：两气覆盖不能只有 `FOLLOW_FORCE`。至少要分出：

- `LEAK_EXCESS`：旺神太过，取泄秀。
- `RESTRAIN_BALANCED_EXCESS`：旺而未太过，取制衡。
- `SUPPORT_ROOTED_WEAK`：弱神有根，扶弱神成用。
- `REMOVE_ROOTLESS_WEAK`：弱神无根，去其寡。
- `FOLLOW_CLEAR_BODY`：清气成体段、两气同心或均敌成象，且无有效制化冲突，顺其清势。

### 4. “可顺不可逆”有条件

《滴天髓阐微·疾病》有木火同心例：

```text
此造木火同心，可顺而不可逆，反以壬水为忌。
```

这说明两气同心、气势成体段时，逆向制衡可能反成忌。但这不是所有两气都可顺。它依赖：

- 木火同心；
- 水不能成为有效调候/制化用神；
- 逆水会破坏原局气势。

产品含义：`可顺` 必须是判断结果，不是两气候选的默认结果。

### 5. 金白水清属于“子母和会/清纯”，喜忌有清浊边界

《神锋通考》说：

```text
秋月之水，母旺子相，表里光莹，遇金助则子母俱和，而金白水清。
谓秋金旺，金能生水，则水更遇相会，则曰子母和会，而金白水清也。
```

又有：

```text
水清金白助格中，若见火土多周折。
```

产品含义：金白水清不是“金水 top2”本身，而是秋金生水、子母和会、清而不浊；火土是否破象要看是否混浊或制水破清。

### 6. 火土、土金等组合也可能是泄秀，不一定顺势

《神锋通考》有己土未月例：

```text
土旺喜金宜泄气，养金喜水定超群。
己土生临未月，火土炎蒸……透也酉金，己土则泄秀气在金上去。
以丙丁火为损金之神。
```

产品含义：火土/土金相关形象不能一概顺火土。土旺有时喜金泄秀，金再喜水；火反而可能损金。若产品只按“火土成势 => 喜火土”，会错过“泄秀”型两气。

### 7. 众寡理论是两气/多气取法的一种特殊分支

《滴天髓阐微·众寡》说：

```text
强众而敌寡者，势在去其寡；强寡而敌众者，势在成乎众。
```

并进一步说明，需要“须分日主四柱两端而论”。同篇也给出相反例：

```text
丙火生于九月……坐阳刃会火局，谓之强寡。
年干癸进气，癸水通根余气丑土，泄其火局，庚金生助壬癸为众也。
势在成乎众……
一交己未，火土并旺，父母双亡；及戊午二十年，破败家业。
```

产品含义：`强众敌寡` 不能只看“强党 + 弱敌”。还要判断到底是“强众敌寡，去其寡”，还是“强寡敌众，成乎众”。有根进气的弱敌可能并非该去，而是要扶成众。

### 8. 壬水在夏火土局中可能是相济，不应机械判逆势

《滴天髓阐微·天干》论壬水：

```text
生于九夏，则巳、午、未、申火土之气，得壬水熏蒸而成雨露，故虽从火土，未尝不相济也。
生于四、五、六月，柱中火土并旺，别无金水相助。火旺透干则从火，土旺透干则从土，调和润泽，仍有相济之功也。
```

产品含义：夏火土旺时，壬水不一定是破势；若通根或有金水相助，可能是相济/调候/制刃关键。只有“别无金水相助”并且火土真成势时，才更接近从火从土。

### 9. 任铁樵案不是孤例式例外，而是“弱神有根宜扶/相济不成”的边界

《滴天髓阐微·官杀》任铁樵自造：

```text
壬水虽通根身库，总之无金滋助，清枯之象。
兼之运走四十载木火，生助劫刃之地……
至卯运，壬水绝地，阳刃逢生……倾家荡产。
```

理论含义：

- 壬水不是无根弱神，所以不能去。
- 但壬水无金滋助，救应不足，所以为清枯。
- 木火运继续助刃，为忌。

产品含义：任铁樵不能抽象成“火土成势 veto 的特例”，而应归入更通用的“有根制衡神不可被顺势误杀；但制衡神不足时，应标为清枯/救应不足”的模型。

### 仅显示 TWO_QI_IMAGE、但未覆盖的 case

这类 case 的 `image_subtype` 可能显示为 `火金成象`、`木土成象`、`水木成象` 等，但 `decision_chain` 没有 `L0 特殊气势`，实际仍走调候、格局、扶抑等普通链路。

代表 case：

- 袁树珊：火金成象，但仍按七杀格、调候、扶抑评分。
- 文天祥：火金成象，但仍按建禄格、财官调候评分，结果通过。
- 戴笠：火金成象，但仍按火旺无水调候评分，结果基本正确。
- 刘墉：木土成象，但仍按正官格春土用丙评分，结果通过。
- 戚继光：水土成象，但仍按调候火印评分，结果用神命中。

因此问题不是“出现 TWO_QI_IMAGE 就错”，而是“命名两气触发 L0 覆盖时缺少覆盖前审查”。

## 产品判断原则

### 两气候选的五层判定

`TWO_QI_IMAGE` 应从“是否覆盖”升级为五层判定：

| 层级 | 判断问题 | 产品输出 |
| --- | --- | --- |
| L1 候选 | top2 是否形成明显气势集中 | 仅形成 `TWO_QI_IMAGE` 候选 |
| L2 成象 | 两气关系是相生、相克均敌、同党同心，还是只是 top2 偏聚 | 决定是否进入两气成象候选 |
| L3 清浊 | 是否清气成体段、生化有情、少杂气破局 | 决定是否可进入命名形象 |
| L4 旺弱 | 旺神太过/不太过，弱神有根/无根；两气不均时重心在哪 | 决定泄、克、扶、去、顺 |
| L5 制化 | 是否已有格局制化/调候/救主指定用神 | 决定是否覆盖下层 |
| L6 展示 | 最终是覆盖、降级、还是描述 | 输出策略和解释链 |

### 应该覆盖的场景

满足以下条件时，命名 `TWO_QI_IMAGE` 可以覆盖普通格局和喜忌：

1. **清气成体段**

   两气不是单纯 top2，而是“安放得所、生化有情”，闲神忌客不深混、不司令，或能被岁运/原局制化。

2. **取法确实由两气形象主导**

   例如：

   - 朱元璋：教材按火土成势，不按普通扶抑。
   - 冰心：教材按金白水清，两气清秀。
   - 张謇：教材按强众敌寡、去其寡。

3. **两气关系与旺弱处置符合“泄/克/扶/去/顺”**

   不能默认顺势。必须先判断：

   - 相生两气：看是否母子和会、生化有情；若旺神太过，优先泄秀。
   - 相克两气：看是否均敌；均敌可成象，偏重偏轻则转入重心判断。
   - 旺神太过：宜泄。
   - 旺神不太过：宜克。
   - 弱神有根：宜扶。
   - 弱神无根：宜去。
   - 两气同心且逆神无情：可顺。

4. **覆盖后的喜忌方向与清浊成败一致**

   若覆盖会把有根真神、调候真用、制刃真用打成忌，则不能覆盖。

5. **top2 气势足够集中**

   当前阈值可保留为初版标准：

   - 金白水清：金水合计 >= 0.60，且月令在申/酉。
   - 火土成势：火土合计 >= 0.62。
   - 强众敌寡：火土合计 >= 0.62，且存在弱敌元素。

### 不应该覆盖的场景

以下场景应触发 veto 或降级为 `DESCRIPTIVE_ONLY`：

1. **弱神有根，且承担制化/相济功能**

   按“弱神有根者宜扶”。如果第三气虽弱，但透干有根、能制刃/调候/相济，就不能当作杂气或逆势忌神。

   例如任铁樵：

   - 火土 top2 合计约 70%，看似火土成势；
   - 但壬水透干且通根身库；
   - 格局层已识别羊刃格、刃杀相停；
   - 调候首取壬水；
   - 教材取壬水七杀制劫刃，喜金水，忌木火。

   这时火土成势只能作为“火土偏旺/清枯背景”，不应 L0 顺势覆盖。

2. **旺神太过，应泄不应克，或应泄不应顺**

   例如土旺喜金泄气、养金喜水。此类不能简单归为“土旺继续喜火土”。

3. **普通格局制化比两气形象更具体**

   例如：

   - 七杀格财滋杀；
   - 食神制杀；
   - 杀化印；
   - 合官留杀；
   - 羊刃驾杀 / 刃杀相停。

   如果两气覆盖会吞掉这些具体格局取法，应优先保留格局制化。

4. **众寡方向未判清**

   强众敌寡必须先分“强众敌寡”还是“强寡敌众”。弱敌如果进气通根、且有财印食伤扶助，可能不是要去，而是要成乎众。

5. **古籍稀疏格法 case**

   例如张时佥事：

   - 书中是魁罡格取财官印；
   - 当前被强众敌寡覆盖。

   这类应先进入 `classical_pattern` 独立层，不作为通用两气覆盖收益或失败的主要指标。

6. **两气只是统计现象**

   若 subtype 是 `木土成象`、`火金成象`、`水木成象` 等非命名审核形象，应保持描述性，不覆盖。

## 覆盖决策流程

建议把当前一跳式覆盖改为两阶段：

```text
TWO_QI_IMAGE 候选
  -> 两气关系与均敌/重心判断
  -> 命名形象识别
  -> 清浊/有情评估
  -> 旺弱处置判断
  -> 格局制化/调候冲突检查
  -> 古籍格法 veto
  -> 输出覆盖 / 降级 / 描述
```

### Step 1: 两气关系与均敌/重心判断

新增 `assessTwoQiFormation()`：

```text
输入：top2 元素、两气占比差、月令、透干、根气、第三气
输出：
  relation: GENERATING | CONTROLLING | SAME_CAMP | CHAOTIC
  balance_state: EVEN | SLIGHTLY_BIASED | HEAVILY_BIASED
  center_of_gravity: top1 element | null
  formation_level: FORMED | PARTIAL | STATISTICAL_ONLY
```

基本规则：

- 相生两气：重点看子母和会、泄秀是否有源。
- 相克两气：必须近于均敌；偏重偏轻时不直接成象。
- 同党两气：看是否同心同势，以及逆神是否真无情。
- 两气不均等：记录重心，交给旺弱处置层决定泄、扶、去或顺。

### Step 2: 命名形象识别

保留当前命名形象：

- `金白水清`
- `火土成势`
- `强众敌寡`

普通 `${元素}${元素}成象` 仍为描述性。

### Step 3: 清浊/有情评估

新增 `assessTwoQiPurity()`：

```text
输入：top2 元素、月令、透干、根气、杂气、冲合刑害
输出：
  purity_level: CLEAR | MIXED_BUT_PLACED | TURBID | BROKEN
  relationship: GENERATING | SAME_CAMP | CONFLICTING_BUT_ORDERED | CHAOTIC
  mixed_qi_status: NONE | PLACED | DISTURBING | USEFUL
```

基本规则：

- 生化有情、子母和会、泄秀有源：可升清。
- 杂气不司令、不深藏、不破喜用：可视为安放。
- 杂气透干有根且破喜用：转浊或破象。
- 清气不当权但亲神能扶清、忌客不党浊：可保留但不强覆盖。

### Step 4: 旺弱处置判断

新增 `resolveTwoQiTreatment()`：

```text
旺神太过 -> LEAK_EXCESS
旺神不太过 -> RESTRAIN_BALANCED_EXCESS
弱神有根 -> SUPPORT_ROOTED_WEAK
弱神无根 -> REMOVE_ROOTLESS_WEAK
两气同心且逆神无情 -> FOLLOW_CLEAR_BODY
```

这一层决定取用方向，而不是直接由 subtype 决定。

### Step 5: 覆盖资格初筛

命名形象需要同时满足：

- `allow_override === true`
- `match_score >= reviewed_override_min_score`
- `yongshen_strategy !== DESCRIPTIVE_ONLY`
- `formation_level === FORMED`
- `purity_level` 不是 `TURBID/BROKEN`
- `treatment` 支持覆盖下层

### Step 6: 有效制衡/制刃 veto

`hasEffectiveCounterUse()` 应从“第三气 veto”升级为“真神/亲神保护”。满足以下条件时，命名两气不得直接覆盖：

```text
非 top2 之气或弱势之气透干有根
AND (
  调候首取为第三气
  OR 普通格局制化指向第三气
  OR 救主/制刃结构指向第三气
  OR 该气能使两气从浊转清、从偏枯转相济
)
```

火土成势的重点 veto：

```text
火土为 top2
AND 壬癸水透干有根
AND 原局存在羊刃/刃杀相停/杀制刃信号
=> 不顺火土成势
```

强众敌寡的重点 veto：

```text
火土为 top2
AND 弱敌不是教材/规则中的“去其寡”对象
AND 普通格局已形成财滋杀、食神制杀、杀化印等更具体取法
=> 不按强众敌寡覆盖
```

### Step 7: 古籍格法 veto

如果 case 或规则层识别到 `classical_pattern`，则两气形象默认只展示，不主导喜忌。

适用对象：

- 魁罡格
- 日德格
- 相刑遇贵
- 官印双全
- 印绶天德
- 天干顺食格
- 地支夹拱格
- 两干不杂格

### Step 8: 输出策略

| 判定结果 | 输出 |
| --- | --- |
| 通过覆盖 | `override_normal_pattern: true`，进入 L0 |
| 泄秀/制衡/扶弱优先 | 不进入 L0 顺势；回到普通链路或专用策略评分 |
| 被 veto | `override_normal_pattern: false`，保留 subtype，记录 `override_veto_reason` |
| 普通两气 | 仅展示，不覆盖 |

## 前端与字段呈现 PRD

### 背景问题

当前前端已有“形象/从旺”模块，主要消费 `image_analysis.primary_candidate`、`image_analysis.override_candidate` 与 `override_normal_pattern`。

现有呈现是二元的：

```text
形象判断：某某形象，匹配度xx%，已覆盖常规喜忌 / 暂不覆盖常规喜忌
```

这在旧模型里勉强可用，但在 TWO_QI_IMAGE 修复后不够表达真实推断链。未来两气形象会出现至少四种状态：

1. **已覆盖喜忌/用神**：如冰心、朱元璋、张謇。
2. **仅作形象展示，不覆盖**：如普通 `火金成象`、`木土成象`。
3. **候选成立但被 veto**：如任铁樵火土成势候选，被壬水制刃/调候挡下。
4. **古籍格法优先**：如张时佥事，强众敌寡候选不能覆盖魁罡格取法。

如果前端仍只显示“已覆盖/暂不覆盖”，用户会看到以下冲突：

- 形象显示火土成势，但用神却取壬水；
- 格局显示建禄/羊刃，但喜忌按特殊形象；
- 旺衰显示身弱/身中，但 L0 语义又说顺势成象；
- 候选被 veto 时，用户不知道是模型没识别，还是识别后被更高优先级挡下。

### 产品目标

前端不新增大模块，沿用当前“形象/从旺/形象校验”位置，但展示语义从“是否覆盖”升级为“特殊形象决策”：

```text
识别到什么形象
是否真正成象
采用什么取用策略
覆盖了哪些字段
如果未覆盖，被什么规则挡下
```

### 后端字段契约

在 `image_analysis.primary_candidate` / `override_candidate` 上新增或透传以下字段：

| 字段 | 类型 | 用途 |
| --- | --- | --- |
| `formation_level` | enum | `FORMED` / `PARTIAL` / `STATISTICAL_ONLY`，说明是成象、半成象，还是仅统计偏聚。 |
| `formation_relation` | enum | `GENERATING` / `CONTROLLING` / `SAME_CAMP` / `CHAOTIC`，说明两气关系。 |
| `balance_state` | enum | `EVEN` / `SLIGHTLY_BIASED` / `HEAVILY_BIASED`，说明相克或两气不均时是否均敌。 |
| `center_of_gravity` | string/null | 两气不均时的重心五行。 |
| `purity_level` | enum | `CLEAR` / `MIXED_BUT_PLACED` / `TURBID` / `BROKEN`，说明清浊。 |
| `treatment` | enum | `LEAK_EXCESS` / `RESTRAIN_BALANCED_EXCESS` / `SUPPORT_ROOTED_WEAK` / `REMOVE_ROOTLESS_WEAK` / `FOLLOW_CLEAR_BODY`。 |
| `override_scope` | enum | `none` / `display_only` / `yongshen_only` / `xiji_yongshen` / `full`。 |
| `affected_dimensions` | string[] | 本次覆盖影响的维度，如 `geju`、`yongshen`、`xiji`、`strength_semantic`。 |
| `override_reason` | string | 覆盖成立原因。 |
| `override_veto_reason` | string | 未覆盖或降级原因。 |
| `veto_basis` | string[] | 触发 veto 的规则标签，如 `ROOTED_COUNTER_USE`、`TIAOHOU_TRUE_USE`、`CLASSICAL_PATTERN_PRIORITY`。 |
| `effective_counter_element` | string/null | 保护的第三气或弱神，如 `水`、`金`。 |
| `effective_counter_basis` | string[] | 第三气为何有效：透干、有根、调候、制刃、格局制化、救主等。 |

兼容字段：

- `override_normal_pattern` 继续保留，作为旧前端和旧 prompt 的布尔兼容层。
- 新前端优先读 `override_scope`；缺失时降级到 `override_normal_pattern`。

### 前端展示规范

所有展示规范均服从“断语来源门禁”：状态 chip、算法链路、覆盖范围可以由规则引擎生成；涉及气质、职业、关系、身心、注意事项、岁运吉凶的解释，必须来自 source-backed statement library。没有来源时，显示结构判断，不显示现代断语。

#### 状态 1：已覆盖喜忌/用神

适用：`override_scope` 为 `xiji_yongshen` 或 `full`。

示例文案：

```text
识别为金白水清，两气清纯，取水泄秀；此形象覆盖常规建禄取法，影响格局、用神、喜忌。
```

展示要点：

- chip：`已主导取用`
- 显示影响范围：`格局 / 用神 / 喜忌`
- 显示取用策略：如 `泄秀`、`去寡`、`顺势`

#### 状态 2：仅形象展示，不覆盖

适用：`override_scope = display_only` 或 `none`，且无 veto。

示例文案：

```text
见火金两气偏聚，但未构成命名特殊形象，仍按七杀格、调候与扶抑取用。
```

展示要点：

- chip：`仅展示`
- 不暗示喜忌被形象主导
- 继续展示普通格局/调候链路

#### 状态 3：候选成立但被 veto

适用：有 `override_veto_reason`。

示例文案：

```text
见火土成势候选，但壬水透干通根，承担制刃与调候；不按火土顺势覆盖。
```

展示要点：

- chip：`候选被挡下`
- 展示 veto 原因
- 展示被保护的元素/十神，如 `壬水七杀`
- 明确最终仍按普通格局制化或调候取用

#### 状态 4：古籍格法优先

适用：`veto_basis` 包含 `CLASSICAL_PATTERN_PRIORITY`。

示例文案：

```text
见强众敌寡候选，但书中以魁罡格取财官印，本次按古籍格法优先。
```

展示要点：

- chip：`古籍格法优先`
- 不把 TWO_QI_IMAGE 计为主要失败/成功原因

### 影响范围说明

前端需要显式说明覆盖影响哪些东西：

| 维度 | 当前覆盖行为 | 新呈现要求 |
| --- | --- | --- |
| 格局 | 直接或间接展示为特殊形象 | 显示 `affected_dimensions` 是否包含 `geju`。 |
| 用神 | L0 直接给 top yong | 显示 `treatment` 与 `yongshen` 影响。 |
| 喜忌 | L0 直接生成 favorable/unfavorable | 显示 `affected_dimensions` 是否包含 `xiji`。 |
| 旺衰 | 当前字段不一定改，但语义被 L0 压过 | 若包含 `strength_semantic`，显示“旺衰语义按特殊形象解释”；否则保留原旺衰字段并提示未覆盖。 |

### 修改方案

#### Phase 1：后端字段与兼容层

1. 在 `baziImageAssessor` 的 TWO_QI_IMAGE candidate 中新增 formation/purity/treatment/override 字段。
2. 保留 `override_normal_pattern`，但由 `override_scope` 推导：
   - `full`、`xiji_yongshen` => true
   - `display_only`、`none` => false
3. `serializeCandidate()` 透传新增字段。
4. `BaziRuleEngine.getSpecialPattern()` 不再只看布尔值，改看 `override_scope`：
   - `full`：可覆盖格局、用神、喜忌、旺衰语义；
   - `xiji_yongshen`：只覆盖喜忌/用神，不强改格局；
   - `display_only` / `none`：不进 L0。

#### Phase 2：前端形象/从旺模块改造

1. `BaziStaticPanel.vue` 增加 `imageDecisionViewModel`：
   - label：形象名；
   - statusChip：已主导取用 / 仅展示 / 候选被挡下 / 古籍格法优先；
   - strategyLabel：泄秀 / 制衡 / 扶弱 / 去寡 / 顺势；
   - affectedText：影响格局/用神/喜忌/旺衰语义；
   - reasonText：覆盖或 veto 原因。
2. `BaziView.vue` 的“形象校验”区域同步使用同一套 view model。
3. 对旧数据做 fallback：
   - 有 `override_normal_pattern === true` 但无 `override_scope`：显示 `已主导取用`；
   - 无新增字段：保持旧文案。

#### Phase 3：问答与报告文案

1. 新增 source-backed statement library：
   - 每条语料记录 `source_title`、`source_excerpt/source_summary`、`paraphrase`、`covered_fields`、`applicable_when`。
   - 未绑定来源的条目 `displayable: false`。

3. `baziQuestionCore.js` prompt 中保留完整 `image_analysis`，并增加人类可读的 `image_decision_summary`。
4. `worker/src/index.js` 生成档案 LLM 时传入 `pattern_analysis`、`image_analysis`、`source_backed_statements`。
5. `lib/baziLlmSections.js` prompt 明确：
   - 如果 `override_veto_reason` 存在，不得按该形象重取喜忌；
   - 如果 `override_scope` 是 `display_only`，形象只能作为背景描述。
   - 如果某字段无材料来源，不得生成性格、职业、关系、健康或岁运断语，只能写“材料未提供可展开断语”或省略。

## 非目标

- 不在本轮实现完整古籍格法系统。
- 不把所有两气成象都提升为格局。
- 不用单个历史 case 硬编码喜忌。
- 不改变普通调候、扶抑、病药评分的主结构。

## 风险与边界

1. **veto 过宽会导致朱元璋/张謇回退**

   所以 veto 必须要求第三气同时满足“透干、有根、被下层取法指向为主用神”，不能仅凭第三气占比存在。

2. **veto 过窄会修不了任铁樵**

   任铁樵的关键不是水占比最高，而是壬水透干通根且格局/调候均指向制刃。

3. **康熙需要同步处理格局优先级**

   单纯 veto 强众敌寡后，如果仍按建禄调候甲木，可能只能消除土喜方错误，未必完全命中财滋杀。康熙还需要七杀格/财滋杀取法补强。

4. **古籍格法需要独立层**

   张时佥事这类 case 不能靠 TWO_QI_IMAGE 修好，必须等 `classical_pattern` 层。

## 推荐实施顺序

1. 先新增 TWO_QI_IMAGE 覆盖审计输出：
   - `override_veto_reason`
   - `effective_counter_element`
   - `effective_counter_basis`

2. 再实现最小 veto：
   - 任铁樵型火土成势 + 壬癸透根 + 羊刃/刃杀信号。
   - 康熙型强众敌寡 + 财滋杀/七杀制化信号。

3. 跑 golden eval，确认朱元璋、张謇、冰心不回退。

4. 再决定是否扩大到更多两气覆盖 case。
