# 用神 / 目标十神拾取能力评测

**评测分支**：`preview/bazi-report-restructure`
**评测日期**：2026-06-06
**评测目标**：验证 panel 对用神（`fiveShenProfile.yong`）及喜忌的拾取准确性
**案例来源**：陆致极《八字命理学进阶教程》（以下简称「进阶」）
**复现脚本**：[`scripts/eval-yongshen.mjs`](../../scripts/eval-yongshen.mjs)

---

## 重要前提：被评测的是"规则引擎"而非"LLM"

经代码核查，panel 上的用神/喜忌**不是 LLM 拾取**，而是本地确定性规则引擎
[`lib/BaziRuleEngine.js`](../../lib/BaziRuleEngine.js) → `getFavorableUnfavorable()` 计算，
经 [`lib/baziCore.js`](../../lib/baziCore.js) `buildCompleteBaziDetail()` 汇总进 `bazi_detail.five_shens`，
再由 [`src/views/BaziView.vue:2478`](../../src/views/BaziView.vue) 的 `fiveShenProfile` 渲染。

**含义**：
1. 用神只是四柱的确定性函数，与生日绝对年份、大运无关 → 评测脚本对每个目标四柱反查任一匹配阳历日期即可，引擎输出与 panel 完全一致。
2. 评测结果可 100% 复现，不存在 LLM 随机性。
3. 引擎用神以**十神**表达（如「正印」「偏财」），教材常以**五行/具体干支**表达（如「丙火」「甲木」），对照时需按日主换算。

## 评测口径

| 判定 | 含义 |
|------|------|
| ✅ 吻合 | 用神五行与教材一致，且喜忌大方向一致 |
| ⚠️ 部分 | 用神五行或扶抑方向沾边，但关键取法/喜忌有冲突 |
| ❌ 偏差 | 用神五行与教材相反或落在教材明示的忌神上 |

## C01 — 左宗棠（七杀格·阳刃驾杀）

**来源**：进阶 第一章 p.17/25/27 ｜ 纯用神 ｜ 反查阳历 1812-11-10 丑时

| | 四柱 | 性别 | 日主 | 月令 |
|---|---|---|---|---|
| | 壬申 · 辛亥 · 丙午 · 庚寅 | 男 | 丙火 | 亥月（冬令水旺）|

**【教材】** 七杀格，身弱无食制杀 → **取午火（阳刃／劫财）为用神，寅木（偏印）为喜神；喜木火，忌金水**。调候亦喜木火。逻辑：阳刃驾杀 + 比劫助身。

**【模型】**
- 旺衰/格局：身弱 · 七杀格（成格七杀格）
- **用神：偏印（HIGH）**
- 喜 [] ｜ 忌 [正官] ｜ 仇 [正财]
- favorable：偏印（木）｜ unfavorable：正官（水）、正财（金）
- 用神说明：「调候首取甲，日主身弱需生扶，七杀格（逆用）以食神、印绶、劫财为用，综合取枭印为命局用神。」

**判定：⚠️ 部分**（喜忌大方向对，但用神主星取错——经文献核实非"双解"）
- 喜忌大方向 **忌金水 = 教材完全一致** ✅
- 但用神主星错：教材取**午火阳刃（劫财）**，模型取**偏印（木）**。逐维打分：偏印 = 调50+扶20+格20 = **90**（用神），劫财（午阳刃）= 扶20+格20 = **40**。
- **根因：调候让用神虚高**。丙火亥月调候首取甲木（=偏印），调候 +50 把"喜神偏印"硬顶成 90 分用神，盖过真用神午火阳刃（去掉调候 +50，偏印=40，与阳刃打平）。
- **文献裁决（非双解，有明确高下）**：《滴天髓·体用》"杀强身弱则印绶化杀"，但印刃并存看气势——杀与日主/羊刃紧贴交战（此造金水55%汪洋、水火对峙）时"天战犹自可，**地战急如火**"，文和之印来不及化，**须拔羊刃敌杀**。寅木偏印脱离午火即"水泛木浮"（木仅11.7%虚浮），唯午火阳刃是抗杀主力 → **午火阳刃是"毫无争议第一用神"，偏印是"兼调候+化杀生刃的顶级喜神"**。定论："调候神通常作喜神，不能取代解决生死矛盾的用神；七杀攻身先用刃保命，再用印暖局。"
- 与 C04 镜像：C04 调候火**该上位被压**、C01 调候木**上位过头**——同属引擎「**调候喜/用不分**」系统问题。**列待办。**

## C02 — 冼冠生（日主无根·身弱有印）

**来源**：进阶 第二章 p.45-46 ｜ 纯用神 ｜ 反查阳历 1888-11-15 丑时

| | 四柱 | 性别 | 日主 | 月令 |
|---|---|---|---|---|
| | 戊子 · 癸亥 · 庚寅 · 戊寅 | 男 | 庚金 | 亥月（水旺，金无根）|

**【教材】** 庚金地支全无金根，**日主无根**极弱，必须靠天干 **戊土（偏印）生身救主**。**戊土为唯一用神**；运破印根则倾。

**【模型】**
- 旺衰/格局：身弱 · 食神格
- **用神：比肩（HIGH）**
- 喜 [劫财, 七杀] ｜ 忌 [伤官] ｜ 仇 []
- favorable：比肩、劫财（金）｜ unfavorable：伤官（水）
- 用神说明：「…食神格（顺用）以财星、七杀为用，综合取比肩为命局用神。」

**判定（修复前）：❌ 偏差**
- 教材核心是「**无根之金，比劫亦无根，唯印（土）能生**」，故用神必是偏印。
- 模型却取 **比肩（金）**，且喜神/favorable 全无印星，正好漏掉教材唯一用神。用神五行（金 vs 土）不一致，关键「无根需印」逻辑未捕获。

**【模型·修复后】**（见文末「根因修复记录·印星救主格」，引擎已上线 通关有根闸 + 印星救主格）
- **用神：偏印（土）**｜ favorable：偏印（土）｜ unfavorable：正官（火）、伤官（水）
- 通关有根闸灭掉无根比劫的 +30（金10%且地支无金根，虚浮幻影）；印星救主格识别透干有根之戊土偏印为救主用神，弃官就印。

**判定（修复后）：✅ 吻合** — 用神 **偏印（土）= 教材戊土偏印** 五行一致，忌神含伤官（水＝病）方向对齐。

### 系统性短板（4 个 ❌ 高度同因）

引擎在**正格、身弱取印/取财**的常规盘表现稳健（C05/C06/C08/C10 全中，且置信度 HIGH 名副其实）。
失败全部集中在「**需要推翻月令顺逆、做整体取舍**」的盘：

1. **从格判别"明扶不查根"导致从儿格被卡（C03 三毛）** — **注意：引擎的从格框架是完整的**（[`lib/baziImageAssessor.js`](../../lib/baziImageAssessor.js) 已有 从财/从儿/从势/专旺/两气/化气 的成象打分 + override 机制 + L0 特殊气势覆盖），且对三毛**已正确识别出从儿格、match_score=74.38**。真正卡点是一个判定细节：
   - `getDmSupport()`（baziImageAssessor.js:219）已算出 `branch_same_roots=0`、`is_rootless=true`（三毛盘 癸乙癸甲／未卯未寅，地支全无水根）；
   - 但 `hasVisibleSupport`（baziImageAssessor.js:330）= `exposed_same>0 || exposed_seal>0`，**只数天干是否透出比劫/印，不看这些透出物本身有没有根**；
   - 于是那个**无根虚浮的年干癸**被当作"有效明扶"，①扣 `visible_support_penalty=20`（否则 74.38+20=94.38 ≥ `OVERRIDE_MIN_SCORE`=80 即成象），②在 `allowOverride` 里硬性否决（要求 `!hasVisibleSupport`）。双重否决把已成形的从儿格压回 SUSPECTED，回落常规身弱食神格，用神才取到大忌的金。
   - 性质：**非"缺失"，是 override 落地判定的命理立场问题**。详见下「文献裁决」。**✅ 已修复上线**（见文末「根因修复记录」）：C03 现识别为从儿格、用神食神、忌金，对齐教材。
2. **身弱弃官就印未触发（C07 杜月笙）** — 成格已标「正官佩印格」，用神却仍取旺官（病神）而非印。顺用逻辑没接「官多变杀、身弱当弃官」的修正分支。
3. **调候急用神被扶抑惩罚压制（C04 郁达夫）**〔订正：原写"调候未回灌"有误〕 — 调候**确实参与了打分**（5 维：调候/病药/通关/扶抑/格局）。子月甲木调候首取丁火，**伤官（丁火）拿到全场最高调候分 +50**。但身弱扶抑维度对食伤一律 −20（伤官砍到 30），格局护卫分只发印比 → 比劫（扶20+格20=40）险胜伤官（30），用神落比劫（木）。根因：引擎扶抑维度已有「格局保护神豁免负分」机制，却未把同等豁免给"调候急用神"。**✅ 已修复上线**（见文末「根因修复记录·调候为急」，依《滴天髓》"冬木不受水"分层落地）：C04 现取伤官（火），对齐教材。
4. **通关误判 + 扶抑不分印比，致无根之金被捧为用神（C02 冼冠生）** — 庚金亥月、地支无金根（strength 2.6），教材判「伤官佩印」取**戊土偏印**（戊两透坐寅、寅藏戊长生，印有根，挡水救主）。引擎逐维：
   - **通关 +30 误触发**：规则 `{土,水,bridge:金}` 见土29%、水30%、金10%<15%，判「土水交战派金通关」给比劫 +30。但①通关神（金）**无根**，违《滴天髓·通关》"能胜能补、通根得地"之资格；②土对庚金是印（生身）、水是食伤（泄身），本是绕日主的十神关系，非两五行交战。
   - **扶抑不分印比**：庚金无根，引擎对印、比劫**一律 +20**，未体现《滴天髓·衰旺》"干多不如根重""无根需印"——无根之比劫为幻影、自顾不暇，唯有根之印能生根气。
   - 结果：比肩=通30+扶20=**50** ＞ 偏印=扶20=**20**，用神错落无根比劫。教材正解戊土偏印只拿 20。
   - 性质：**多维联动的系统性问题**（通关查根 + 无根佩印识别），非一两行可净修——实测仅去通关 +30 会让七杀（火·亥月调候）以 35 上位、用神再错，需配套「伤官佩印/无根佩印」结构化判定方能压住调候、抬起印。**本轮未改，列待办。**
5. **格局变通漏判（C09，⚠️）** — 只认月令正印格，未做日支取格，导致财（土）喜忌判反。

> 共性：从格/成象框架**已存在且能识别轮廓**，但 **override 落地条件偏严**（明扶判定不查根、门槛 80）+ **身弱时对旺神（官/食伤）的弃用修正缺失**。置信度 HIGH 未与「成象是否 override 成功」挂钩，导致 C03/C07 这类硬伤仍报高信，对用户误导性最大。
