# 八字双人合盘·朋友关系·五层打分规格

> 理论来源：《渊海子平·论兄弟姊妹》《子平真诠·论刑冲会合解法》《子平真诠·论十干合而不合》
> 工程来源：BaziRuleEngine.js / baziRelationScanner.js / baziStateAssessor.js
> 评分区间：-100% ～ +100%（每层归一化后加权合并）

---

## 总分公式

```
Total = DIM1×0.30 + DIM2×0.25 + DIM3×0.20 + DIM4×0.15 + DIM5×0.10
```

各维度先内部归一化到 [-100, +100]，再乘以权重合并。

| 维度 | 权重 | 说明 |
|------|------|------|
| DIM1 同频指数 | 30% | 最基础的人格共振，决定「能不能聊到一起」 |
| DIM2 义气指数 | 25% | 朋友关系核心——关键时刻能不能指望 |
| DIM3 消耗指数 | **20%** | 消耗是真实代价，权重与同频并重才能体现 |
| DIM4 成长指数 | **15%** | 长期价值，窄于前三项 |
| DIM5 钱财边界 | 10% | 实用维度，窄场景 |

> **注意**：DIM3（消耗指数）来源均为负向专项，区间固定为 [−100, 0]，正向数学上不可达。  
> 因此总分理论上限约为 **+80**（DIM3=0 且其余维度全满时），下限为 **−100**（各维度同时最差）。  
> 权重设计意图：让多数普通关系自然落在「塑料情」及以下，只有消耗极低 + 其余三维同时显著正向才能进「普通朋友」以上，大致实现三七开分布。

## Layer 1 · 日主对位（权重 20%）

### 理论依据

《渊海子平·论兄弟姊妹》：「比肩者，兄弟也。」  
日干代表命主本人，两人日干对照是最直接的人格能量对位。

《子平真诠》：「五阳逢财，五阴遇官，俱是作合，惟是本身十干合之，不为合去。」  
合身（日干被对方日干所合）是最深层的命盘亲密感。

### 识别字段

```js
A.bazi_detail.gans[2]  // A 的日干
B.bazi_detail.gans[2]  // B 的日干
getShen(A.dayGan, B.dayGan)  // A 视角下 B 的十神
getShen(B.dayGan, A.dayGan)  // B 视角下 A 的十神
```

### 打分表（L1 原始分 → 归一化到 [-100, +100]）

| A 视角 B | B 视角 A | L1原始 | 含义 |
|---------|---------|--------|------|
| 比（比肩） | 比（比肩） | +80 | 纯同频，天然懂对方 |
| 劫（劫财） | 劫（劫财） | +25 | 有默契，但会暗中竞争 |
| 食（食神） | 枭（偏印） | +60 | A 单向滋养 B，B 倾向接受 |
| 伤（伤官） | 印（正印） | +55 | A 情感表达滋养 B |
| 枭（偏印） | 食（食神） | +60 | B 单向滋养 A（对称） |
| 印（正印） | 伤（伤官） | +55 | B 情感表达滋养 A（对称） |
| 才（偏财） | 杀（七杀） | -35 | A 控制 B 的资源，B 受 A 压制 |
| 财（正财） | 官（正官） | -20 | A 规范 B，关系偏不平等 |
| 杀（七杀） | 才（偏财） | -35 | B 控制 A 的资源，A 受 B 压制 |
| 官（正官） | 财（正财） | -20 | B 规范 A（对称） |

**特殊加分项（检测是否构成天干五合）**

```js
// 从 BaziRuleEngine.calculateInteractions() 或 baziRelationScanner.scanGanRelations()
// 检查 A.dayGan 与 B.dayGan 是否构成五合
const GAN_HEPAIRS = { 甲: '己', 乙: '庚', 丙: '辛', 丁: '壬', 戊: '癸', ... }
if (GAN_HEPAIRS[A.dayGan] === B.dayGan) → 日干五合
```

| 场景 | 额外加分 |
|------|---------|
| A.dayGan 与 B.dayGan 构成天干五合 | +20 |
| 五合可化（月令或天干满足化神条件） | 再 +10 |
| 五合被第三干间隔（合而不合） | 降为 +8 |

## Layer 2 · 日支 × 全局地支交互（权重 25%）

### 理论依据

《子平真诠·论刑冲会合解法》：  
「三方为会，**朋友之意**也。并对为合，**比邻之意**也。」  
日支为关系宫，两人日支相互看是朋友缘分的第一宫位依据。  
「刑冲俱非美事，而三合六合，可以解之。」

### 识别字段

```js
A_dayZhi = A.bazi_detail.zhis[2]
B_dayZhi = B.bazi_detail.zhis[2]
A_allZhis = A.bazi_detail.zhis  // [年支, 月支, 日支, 时支]
B_allZhis = B.bazi_detail.zhis

// 交叉关系计算（来自 baziRelationScanner.scanZhiRelations 或 calculateInteractions）
crossRelations = scanCrossZhis(A_allZhis, B_allZhis)
```

### 2-A 日支对位（占 Layer 2 的 60%）

| 日支关系 | 原始分 | 说明 |
|---------|--------|------|
| 六合（如子丑、卯戌） | +80 | 最自然的贴近感，越相处越舒服 |
| 三合成局（日支参与）| +70 | 缘分型，气场高度融合 |
| 三合半合（两支中的两个）| +45 | 有共鸣，感觉相似 |
| 暗合（地支暗合）| +35 | 隐性吸引力，不明显但存在 |
| 同支（相同地支）| +50 | 生命节奏相似 |
| 无关系 | 0 | 中性 |
| 破 | -25 | 表面平静，暗中消耗 |
| 害 | -40 | 相处有隐性伤害感，说不清的不舒服 |
| 刑（含自刑）| -55 | 容易触发对方的问题，摩擦明显 |
| 冲 | -80 | 关系宫对冲，相处有强烈碰撞感 |

> 注意：「合而不合」情形（两支中间有第三支参与形成更强的组合打断原合）需要通过 `crossRelations` 动态计算，不能只看两支本身。

### 2-B 年支对位（占 Layer 2 的 20%）

年支代表成长背景、家庭环境、童年烙印。  

| 年支关系 | 原始分 | 说明 |
|---------|--------|------|
| 同支 | +55 | 出生年相同或相近，成长背景高度相似 |
| 六合 | +50 | 相性好，聊原生家庭有共鸣 |
| 三合 | +40 | 价值观底层有相似之处 |
| 冲 | -45 | 成长背景差异大，可能有根本性的世界观摩擦 |
| 刑害 | -25 | 有时会无意间戳到对方的痛点 |

### 2-C 全局跨盘地支计分（占 Layer 2 的 20%）

扫描 A 全部 4 支 × B 全部 4 支，共 16 对关系：

```js
let crossScore = 0
for (azhi of A_allZhis):
  for (bzhi of B_allZhis):
    rel = getZhiRelation(azhi, bzhi)
    weight = getPillarWeight(azhi.pillarIndex) * getPillarWeight(bzhi.pillarIndex)
    // 柱位权重：月支 1.4 > 日支 1.3 > 时支 1.0 > 年支 0.8
    crossScore += REL_SCORE[rel] * weight
```

| 关系类型 | 单对基础分 |
|---------|-----------|
| 六合 | +12 |
| 三合/三会 | +15 |
| 暗合 | +8 |
| 冲 | -15 |
| 刑 | -12 |
| 害 | -10 |
| 破 | -6 |
| 无 | 0 |

最终 crossScore 归一化到 [-100, +100]（设定 ±60 为上下边界映射到 ±100）。

## Layer 3 · 全局十神互见（权重 25%）

### 理论依据

《渊海子平·论兄弟姊妹》：  
「甲乙寅卯既多，则兄弟姊妹夺财不和，争斗是非。」——劫财过旺则争夺。  
「见庚则克兄，见辛则伤弟。」——官杀克比肩，朋友受压制。

朋友关系的十神解读：比肩=同频，食神=共同语言+欢乐，印=支持保护，劫财=竞争消耗，七杀=冲突压制。

### 识别字段

```js
// A 视角看 B 的全部 8 个字（天干 + 地支主气），求十神
A.bazi_detail.gans[2]          // A 日干
B.bazi_detail.pillars           // B 全部 4 柱 {gan, zhi, hidden_stems}

// 对每个 B 柱：
getShen(A.dayGan, pillar.gan)          // B天干 → A视角十神
getShen(A.dayGan, pillar.hidden_stems[0])  // B地支主气 → A视角十神
// 注意：空亡支（pillar.is_kong=true）的地支主气权重 ×0.3
```

### 十神友情权重

| 十神（A 视角 B） | 天干单字权重 | 地支主气权重 | 友情含义 |
|----------------|------------|------------|---------|
| 比（比肩） | +18 | +14 | 同频共鸣，天然懂你 |
| 劫（劫财） | -12 | -10 | 暗中竞争，情绪消耗 |
| 食（食神） | +22 | +17 | B带给A欢乐、陪伴、创意输出 |
| 伤（伤官） | +10 | +8 | B表达活跃，能聊天，但有时嘴毒 |
| 财（正财） | +5 | +4 | B有资源，但A需要付出规范成本 |
| 才（偏财） | +12 | +9 | B带来资源流动，相对轻松 |
| 官（正官） | +4 | +3 | B对A有规范性影响，偏正式距离感 |
| 杀（七杀） | -18 | -14 | B对A有压制/冲击感，容易消耗 |
| 印（正印） | +18 | +14 | B保护、照顾、支持A |
| 枭（偏印） | +12 | +9 | B精神支持A，但有时太沉默 |

**柱位加权系数**:

```js
PILLAR_WEIGHT = { 年: 0.8, 月: 1.4, 日: 1.5, 时: 1.0 }
// 日支额外 ×1.5 是因为日支是关系宫
```

**双向计算**:

```js
A_sees_B_score = Σ (getShen(A.dayGan, B_stem_or_maingas) × weight × pillarWeight)
B_sees_A_score = Σ (getShen(B.dayGan, A_stem_or_maingas) × weight × pillarWeight)

L3_raw = (A_sees_B_score + B_sees_A_score) / 2
```

归一化：设 ±120 为满分边界，映射到 [-100, +100]。

**特殊标记（供文案使用，不影响主分）**:

| 场景 | 标记 |
|------|------|
| A看B劫财数量 ≥3 | `⚠ 比劫过旺：财务/资源竞争风险高` |
| B看A七杀在月/日 | `⚠ 七杀冲身：B对A有明显压制感` |
| 双向食神/伤官占多数 | `✓ 食伤共鸣：聊天玩乐天然对频` |
| 双向印星占多数 | `✓ 印星互护：互相都有保护照顾的意愿` |

## Layer 4 · 五神互补（权重 20%）

### 理论依据

陆致极《八字命理动态分析教程》：原局喜用的直接满足即静态贵人。  
用神 > 喜神 > 闲神 > 仇神 > 忌神，是五神对命盘影响的优先级。  
一方命盘中旺的五行，是另一方的用/喜神 → 真贵人；激活忌/仇神 → 真消耗。

### 识别字段

BaziProfile 已经通过 `getFavorableUnfavorable` 算出完整的五神分层，直接取用：

```js
// A 的五神（十神口径）
A.bazi_detail.favorable.five_shens.yong   // 用神：'食神'
A.bazi_detail.favorable.five_shens.xi     // 喜神列表：['正印']
A.bazi_detail.favorable.five_shens.ji     // 忌神列表：['七杀']
A.bazi_detail.favorable.five_shens.chou   // 仇神列表：['偏财']
A.bazi_detail.favorable.five_shens.xian   // 闲神列表：['比肩', ...]

// A 的五神对应五行（已算好）
A.bazi_detail.favorable.wuxing.favorable   // 用+喜的五行：['火', '水']
A.bazi_detail.favorable.wuxing.unfavorable // 忌+仇的五行：['金']

// B 的五行分布
B.bazi_detail.favorable.wuxing_ratio       // {木:25, 火:30, 水:15, 金:20, 土:10}
```

### 计算方法

将 A 的五神转成五行权重系数，逐一与 B 的 `wuxing_ratio` 做加权：

```js
// 五神权重系数（决定对方该五行对我的正负影响力）
const SHENSHEN_WEIGHT = {
  yong:  +1.5,   // 用神：最缺、最急需
  xi:    +1.0,   // 喜神：次要需求
  xian:   0.0,   // 闲神：无影响
  chou:  -1.0,   // 仇神：间接消耗
  ji:    -1.3,   // 忌神：直接消耗
}

function getWuxingForShen(dayGan, shenName) {
  // 用 getWuxingByShenFull(dayGan, shen) 转成五行
}

// 构建 A 的五行→权重映射
function buildWxWeightMap(A) {
  const map = {}
  const fs = A.bazi_detail.favorable.five_shens
  for (const [tier, coeff] of Object.entries(SHENSHEN_WEIGHT)) {
    const shens = tier === 'yong' ? [fs.yong] : fs[tier]  // yong 是单个
    for (const shen of shens) {
      const wx = getWuxingForShen(A.dayGan, shen)
      // 同一五行可能被多个十神占据，取最高权重
      map[wx] = Math.max(map[wx] ?? -Infinity, coeff)
    }
  }
  return map  // e.g. {火:+1.5, 水:+1.0, 金:-1.3, 木:0, 土:-1.0}
}

// A 从 B 命盘里得到的互补分
function getCompScore(A, B_wuxingRatio) {
  const wxMap = buildWxWeightMap(A)
  let score = 0
  for (const [wx, coeff] of Object.entries(wxMap)) {
    score += (B_wuxingRatio[wx] ?? 0) * coeff
  }
  return score  // 原始值，例如 +38 或 -22
}

A_gets = getCompScore(A, B.wuxing_ratio)
B_gets = getCompScore(B, A.wuxing_ratio)
L4_raw = (A_gets + B_gets) / 2
```

归一化：设 ±65 为满分边界，映射到 [-100, +100]。

### 结果场景解读（供文案参考）

| 场景 | 文案方向 |
|------|---------|
| A_gets > +40 且 B_gets > +40 | `双向滋养：在一起互相补给，越来越好` |
| A_gets > +30，B_gets < +10 | `单向贵人：TA是你的贵人，但你补不到TA` |
| A_gets < -20 且 B_gets < -20 | `双向消耗：见面就漏电，越近越累（敌蜜象）` |
| A_gets > +20，B_gets < -20 | `不对等消耗：你在吸TA的能量（或反之）` |

## Layer 5 · 月柱社交宫 + 神煞（权重 10%）

### 理论依据

月支是命盘的社交宫、职业宫，代表此人在公共场合中的气场和处世方式。  
《子平真诠》：「月令为用神之本。」朋友关系中月支对位直接影响「是否在同一社交圈」。

神煞是传统命理对人际关系的快速识别机制，桃花/天乙贵人/文昌为亲和力神煞。

### 识别字段

```js
A_monthZhi = A.bazi_detail.zhis[1]    // A 月支
B_monthZhi = B.bazi_detail.zhis[1]    // B 月支

A_shensha = A.bazi_detail.shensha     // {year: "桃花 天乙", month: "文昌", day: "...", time: "..."}
B_shensha = B.bazi_detail.shensha

// 神煞检测（来自 BaziRuleEngine.calculateShenShaFull）
// 关键神煞：桃花、天乙贵人、文昌、天喜、红鸾、孤辰、寡宿、羊刃、元辰
```

### 5-A 月支社交宫对位（占 Layer 5 的 55%）

| 月支关系 | 原始分 | 含义 |
|---------|--------|------|
| 同支 | +75 | 出生在同一月令，社交节奏完全一致 |
| 六合 | +65 | 社交风格互补且舒适，容易在同一圈子里 |
| 三合（两支中两个）| +50 | 有共同的社交场或行业圈 |

扫描 A、B 各自神煞（shensha 字符串拆分），查找如下匹配：

| 神煞场景 | 原始分 | 说明 |
|---------|--------|------|
| A 的桃花落在 B 的日支或年支 | +22 | A 天然对 B 有亲近感 |
| B 的桃花落在 A 的日支或年支 | +22 | 反之 |

| A 有孤辰/寡宿在 B 的命位 | -18 | 相处有莫名孤独感，即使在一起也不接地气 |
| A 有羊刃，B 日支为 A 羊刃位 | -22 | 容易在关键时刻产生激烈冲突 |

| A 有德秀贵人（月支组）落 B 月干 | +15 | 在职业社交圈里互相欣赏 |

## 完整计算示例

- B：壬日干，日支子，月支亥，喜用五行=[木,火]，忌神=[土,金]，wuxing_ratio={水:40,木:25,火:15,金:10,土:10}

```

L2:
  日支：卯子 → 卯子刑 → -55

  全局支交叉：亥卯未半合 +45, 子寅无关 0, ...

  A看B：壬是甲的枭（偏印）+12×月支权重1.4...

L5:
  月支：寅亥六合 → +65
  神煞：需要实际 shensha 字段...
```

## 关系类型判定规则

> 由于 DIM3（消耗指数）恒 ≤ 0，总分理论上限约 +80，以下阈值已据此校准。  
> 设计意图：多数普通组合自然落在「塑料情」附近，进入「普通朋友」以上需要消耗极轻 + 其余维度显著正向。

| 总分区间 | 判定标签 | 文案风格 |

## DIM2 义气指数 · 文本枚举规格

> **核心机制**：义气指数由两层叠加——印星（印/枭）决定「想不想帮」，L4 五行互补决定「帮了有没有用」。两层都有才是真贵人；有心无力是另一种，有力无心是另一种。文案必须写清楚这个区别，不能混成一个模糊的「支持感」。
>
> **文案原则**：写帮忙那件事本身——介绍、出面、搭把手，还是帮了反而更乱。不写关系「多好」，写实际发生什么。
>
> **语气规则：高分夸张，让人觉得这种命配可遇不可求；低分要毒，说清楚那个帮不上、帮了也没用的真相。**

### 得分构成

```
DIM2 = L3_印星分（印/枭十神，双向加权） + L4_互补分（五行补给方向归一化）
```

- **L3 印星**：「想帮」的底层本能。印星密 = 见了对方就有护着的冲动，会主动出面。
- **L4 五行互补**：「帮了有没有用」。对方的五行旺度正好是你的用/喜神 → 找TA办事，事情就是顺；反之激活忌神 → 在一起做事反而推不动。

DIM2 可以真负分（L4 双向消耗拉低至 −50 以下），五个 Badge 档全部覆盖。

## DIM3 消耗指数 · 文本枚举规格

> **核心机制**：消耗指数由两层叠加——L3 劫煞（劫财·七杀·比肩）决定「命盘里有没有写好能量漏点」，L4 忌神激活决定「在一起时是否触发对方命盘的薄弱区」。两层叠负才是真消耗；单层偶有是常态；两层都轻才有充电可能。
>
> **文案原则**：写「在一起之后你的状态」，不写「关系好不好」。聚焦能量层面的物理感受：不扣血 / 慢慢耗 / 猛烈耗。禁止出现「充电」「越来越好」等正向措辞——消耗指数最好的结果是「不额外多耗」，而非正向增益。
>
> **语气规则：最轻档（基本不耗）用中性语气，强调「不损耗」而非充电；最重档（相互摧毁，≤−86）要说透，不留情面但不评判人；中段平实，承认问题但不过分渲染。**
>
> **得分说明**：DIM3 为纯负向专项指标，区间固定为 [−100, 0]，正向数学上不可达。0 = 无消耗（最好结果），−100 = 极度相互消耗。负分段意味着命盘写有结构性消耗，不可通过「努力相处」消除。

## DIM5 钱财边界 · 文本枚举规格

> **核心机制**：钱财边界由两层叠加——L3 劫财/财星占比决定「命盘里有没有写好互相吸财或劫财的结构」，L2 日支冲害决定「两人的社交/资源风格是否正面碰撞」。两层都正向才是合伙最佳；两层都负才是钱不共管；单层有问题是结构性风险，需要明确边界。
>
> **文案原则**：写「合在一起钱会怎样」，不写「TA是不是财神」。聚焦钱财流动的物理感受：财路相开 / 各自独立 / 隐约摩擦 / 混了就出问题。
>
> **语气规则：高分（≥+61）大胆夸合伙可以，让人感受到命盘写好了财运共振；深负（≤−36）要直说，钱不共管是命盘给的建议不是评判；中段务实，告知边界怎么划。**
>
> **得分说明**：正方向 = 在一起财路更顺或互开机会；负方向 = 混钱容易出摩擦，各管各的反而更好。

---

### 得分构成

```
DIM5 = L3_劫财财星分（劫财密度负向 + 财星互补正向） + L2_日支冲害分（冲 / 害 / 无）
```

- **L3 劫财/财星**：扫命盘里对对方有「分夺财气」（劫财）或「产出财气」（财星互补）方向的十神分布。劫财密度高 → 负分；财星指向对方 → 正分。
- **L2 日支冲害**：日支是社交宫，也是对外资源的接口。日支相冲（-55）表示资源风格正面碰撞；日支相害（-30）是隐性摩擦；无冲害为中性。


**文案枚举（4条，按 best 维度触发）：**

| 触发 DIM | 标题 | 正文（25～40字） |
|---------|------|----------------|
| DIM1 同频 | 频道对了 | 说话不用翻译——在一起不会有「聊不下去」的时刻。 |

#### 卡 R3-B：「要留意的地方」

**决策逻辑（两步，DIM3 优先）：**

```
步骤一：先看 DIM3

  if DIM3 ≤ −65:
    触发 → "高消耗"
  elif DIM3 ≤ −38:
    触发 → "慢消耗"
  else:  // DIM3 > −38，消耗尚在可控范围
    进入步骤二

步骤二：从正向维度里找最弱项

  worst = min(DIM1, DIM2, DIM4, DIM5)

  if worst ≥ +20:
    // 四维都还不错，无需专门警告，改为「均衡型」提示
    触发 → "相对均衡"
  else:
    按 worst 所属维度触发对应文案

平局处理：若多个维度同为 worst（差值 ≤5），
  优先级：DIM2 > DIM1 > DIM4 > DIM5
  （义气最容易踩坑，优先提示）
```

**文案枚举（7条）：**

| 触发条件 | 标题 | 正文（25～40字） |
|---------|------|----------------|
| DIM3 ≤ −65 | 相处是有代价的 | 不是感情不好，是气场层面真的在漏电——要给对方留出独处时间。 |
| DIM3 −38 ～ −65 | 高频相处会慢慢察觉 | 频繁见面后各自需要恢复时间——这是命盘结构，不是对方的问题。 |
| worst=DIM1 | 同频程度有限 | 深聊到某个地方会停下来——当玩伴很好，不必强行往深处走。 |
| worst=DIM2 | 义气靠后天 | 命盘没写「会主动帮」——真遇到事，还是自己先想好备选方案。 |
| worst=DIM4 | 成长各靠自己 | 在对方身边做事不会特别顺——关系的价值不在「互相推一把」这件事上。 |
| worst=DIM5 | 钱的事先说清楚 | 涉及钱容易有摩擦——不是不义气，是命盘这层需要明确边界。 |
| 四维均 ≥ +20 且 DIM3 > −38 | 这段关系比较均衡 | 没有特别突出的短板——命盘这层没给太多警告，剩下的靠相处方式。 |
